        Hi Jon,
        This is controlled by the supportUnpublished attribute in ExamineSettings.config.

        By default the InternalIndexer includes unpublished nodes and the ExternalIndexer does not.

        This is so you can search for unpublished content via the backoffice search (which uses the InternalSearcher).

                Bobobrowse targets a different layer in the stack, but there is a nuget package called "Lucene.Net Contrib" which includes a highlighter among other things.

                You need a fair amount of boilerplate code to use it however, I've already been through that pain. Tada: https://gist.github.com/abrigg...

                You'll also need to strip the HTML from the content being indexed or else your highlighting will be a mess. Handily, I've also done this already: https://gist.github.com/abrigg...

                And to configure it in Examine looks like so:
                <add name="ExternalIndexer" type="UmbracoExamine.UmbracoContentIndexer, UmbracoExamine" analyzer="Search.HTMLStripAnalyzer, Search"/>

                                Hi Paul & Elwyn,

                                Yes, you call the highlight code from your search result page. Something like:

                                var highlightService = new LuceneHighlightService();
                                @foreach (var resultNodeId in resultNodeIds)
                                {
                                var node = Umbraco.Content(resultNodeId);
                                var summary = highlightService.GetHighlight(node.BodyText, "ExternalSearcher", "bodyText", LuceneHelper.SanitiseUserQuery(searchText));
                                <li>@node.Name

                                @Html.Raw(summary)
                                </li>
                                }

                                I don't use Umbraco's dynamic stuff so not sure on the use of node.BodyText vs node.bodyText. One of them will work :)

                                                                                                      Antony Briggs Ismail Mayat • 2 years ago

                                                                                                      Hi Ismail,

                                                                                                      Yep, the default analyser for the External index will tokenise quite heavily, including lower casing, word splitting and I think even word stemmming (e.g. "fishing", "fished" to the root word "fish").

                                                                                                      You can either create a separate Examine index for your facets that use the Lucene.Net.Analysis.KeywordAnalyzer but this won't help in the example given in this article where you are offering a keyword search + faceting to the user.

                                                                                                      In that case you need to implement a LuceneIndexer.DocumentWriting event handler and add the field you want to facet on as NOT_ANALYZED e.g.:

                                                                                                      e.Document.Add(new Field(fieldName, value, Field.Store.YES, Field.Index.NOT_ANALYZED));

                                                                                                      Having used raw Lucene in the past, I was pleasantly surprised at how simple Examine makes this stuff!
